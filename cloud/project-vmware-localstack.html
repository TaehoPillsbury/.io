LocalStack 기반 로컬 클라우드 실습 프로젝트

2025 | 로컬 학습 환경

프로젝트 목표

로컬 PC와 VMware 기반 가상환경에서 LocalStack을 활용해 AWS와 거의 동일한 클라우드 환경을 재현하고, S3·Lambda·API Gateway·DynamoDB·IAM 등 주요 AWS 서비스를 비용 없이 실습할 수 있는 로컬 클라우드 플랫폼을 구축하는 것입니다.

LocalStack 기반의 로컬 클라우드 인프라를 구축하여 API Gateway · Lambda · DynamoDB · S3 등 AWS 서버리스 아키텍처를 실제 AWS 없이 완전하게 구현하고 테스트할 수 있는 실무형 개발 환경을 완성했습니다.

전체 아키텍처

구성 요소

  • LocalStack (컨테이너): AWS API Gateway, Lambda, S3, DynamoDB 등 Full AWS API 에뮬레이션
  • LocalStack Client (컨테이너): awscli / terraform / curl 테스트 전용
  • 로컬 테스트 웹페이지 (S3 정적 호스팅): JS에서 Lambda + API Gateway 호출
  • Lambda 함수 (Python): DynamoDB Local에 데이터 저장, Kubernetes 클러스터 조작(옵션) 등 복잡한 기능도 로컬에서 테스트
  • DynamoDB Local 테이블: 웹 애플리케이션 사용자 데이터 저장
  • API Gateway: S3 정적 웹페이지와 Lambda 연결

VMware 환경

  • 노트북에 VMware 17.5 Pro 설치 및 라이센스 구성
  • Linux 가상머신 생성 및 네트워크 설정
  • Docker Compose로 LocalStack + Client 환경 구성
  • 포트 매핑 및 서비스 활성화 (S3, Lambda, DynamoDB, APIGW 등)

주요 구현 기능

1. LocalStack 클러스터 구축

  • Docker Compose로 LocalStack + Client 환경 구성
  • 포트 매핑 및 서비스 활성화 (S3, Lambda, DynamoDB, APIGW 등)
  • AWS CLI endpoint를 LocalStack으로 변경하여 테스트
  • aws --endpoint-url=http://localhost:4566 s3 ls 명령어로 LocalStack S3 접근

2. S3 정적 웹사이트 호스팅

  • 웹파일(index.html, script.js 등)을 LocalStack S3에 업로드
  • AWS S3와 동일한 방식으로 버킷 정책, 정적 호스팅 설정
  • 웹에서 API Gateway 호출 → Lambda → DynamoDB 로직 구현

3. API Gateway + Lambda + DynamoDB 연동

  • Lambda(Python 3.9) 함수를 ZIP 배포
  • API Gateway REST API 생성
  • /create-user, /create-container 같은 실제 API 구조 설계
  • DynamoDB Local에 데이터 저장/조회 구현
  • 예: table.put_item(Item={"userId": "user01", "score": 100})

4. 컨테이너 기반 Lambda 실행 (Advanced)

  • Lambda 안에서 Kubernetes API 호출
  • 로컬 k3s 클러스터와 Integration (Pod 생성, 삭제)
  • kubeconfig.yaml을 Lambda 패키지 내부에 포함
  • LocalStack이 제공하는 lambda container-mode 사용

5. Terraform 연동 테스트

  • LocalStack provider를 이용해 Terraform apply → 로컬 AWS 리소스 생성
  • 실제 AWS와 동일한 방식으로 IaC(Infrastructure as Code) 실습 가능
  • Terraform 코드로 S3, Lambda, DynamoDB, API Gateway 등 모든 리소스 생성

프로젝트에서 얻은 역량

1. AWS 서비스 아키텍처의 핵심 원리 이해

실제 AWS 없이도 S3·Lambda·APIGW·DynamoDB·IAM 등 핵심 서비스의 구조를 완전히 체험할 수 있었습니다.

2. 서버리스(Serverless) 아키텍처 설계

API Gateway → Lambda → DynamoDB로 이어지는 서버리스 패턴을 직접 구현하며 서버리스 아키텍처의 설계 원칙과 실제 적용 방법을 학습했습니다.

3. IaC (Terraform) 실무 경험

Terraform을 LocalStack에 적용하며 선언형 인프라 관리 방식을 이해하고, 실제 AWS와 동일한 방식으로 인프라 코드를 작성하고 배포하는 경험을 쌓았습니다.

4. 애플리케이션-인프라 통합 능력

정적 웹 → API → Lambda → DB 전체 파이프라인을 로컬에서 완성하며, 프론트엔드부터 백엔드, 데이터베이스까지 전체 시스템 아키텍처를 설계하고 구현할 수 있는 능력을 향상시켰습니다.

5. 비용 절감 전략

AWS 비용 발생 없이 충분히 실무형 아키텍처를 테스트할 수 있는 환경을 구축하며, 클라우드 비용 최적화와 효율적인 학습 방법론을 체득했습니다.

기술 스택
