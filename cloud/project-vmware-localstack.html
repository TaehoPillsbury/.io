LocalStack + MinIO + Cloudflare + Kubernetes 자동 생성 플랫폼 구축

2025.10.1~31 | VMware Workstation Pro 17.5 | 로컬 환경에서 가상 AWS 서비스를 활용한 Kubernetes Pod 자동 생성 프로젝트

프로젝트 개요

로컬 환경(VMware Pro 17.5)에서 실제 AWS 아키텍처와 동일한 방식으로 동작하는 클라우드 자동 배포 플랫폼을 구축한 사례입니다.

LocalStack·MinIO·Cloudflare·k3s를 통합하여 서버리스 + 컨테이너 + 객체 스토리지 + 네트워크 보안 + 로드밸런싱을 모두 포함하는 풀스택 클라우드 환경을 구현했습니다.

해당 플랫폼은 사용자가 웹(UI)을 통해 요청하면 자동으로 Kubernetes에 Deployment를 생성하며, 생성된 리소스는 DynamoDB(LocalStack)에 로그로 저장됩니다.

또한, Cloudflare로 외부 보안 레이어를 구축하여 로컬 서버를 외부에 노출하지 않고도 안전한 접근이 가능하도록 설계했습니다.

시스템 아키텍처 및 구성 요소

1. 사용자(User)

  • 웹 브라우저를 통해 플랫폼에 접근하며, Cloudflare 인증을 통과한 사용자만 내부 리소스에 접근 가능

2. Cloudflare Tunnel (Zero Trust Access)

  • 내부 서버를 외부 인터넷에 직접 노출하지 않고도 안전하게 접근 가능
  • HTTPS 터널 기반 보안 강화
  • 사용자 인증 (Google, GitHub 등 SSO 연동 가능)
  • Cloudflare는 전체 인프라의 Zero Trust Gate 역할을 수행하여, 로컬 VMware 환경에서도 엔터프라이즈 수준의 보안을 확보함

3. WEB Server (HAProxy)

  • 정적 웹 페이지 및 사용자 요청을 받아 템플릿 렌더링
  • 로컬 환경에서 API Gateway 또는 LocalStack 서비스로 프록시 역할 수행
  • 스케일 아웃이 가능한 가벼운 웹 서버 구조

4. MinIO (S3 Object Storage 대체)

  • 로컬 환경에서 S3 오브젝트 스토리지를 완전히 대체
  • 포트: 9000(API), 9001(Console)
  • 주요 역할:
  • - 정적 웹 사이트 호스팅
  • - Lambda 패키지 저장
  • - 배포 로그/결과 백업 저장소
  • AWS S3와 100% API 호환되므로 boto3, AWS CLI로 그대로 접근 가능

5. LocalStack (AWS 클라우드 환경 재현 플랫폼)

  • 포트: 4566
  • Lambda, API Gateway, DynamoDB, IAM, SQS 등 모든 AWS 서비스가 로컬에서 실행
  • AWS 비용 없이도 실제 AWS 구조와 동일하게 개발/테스트 가능
  • 구성 요소:
  • Lambda: API Gateway 요청을 받아 Kubernetes(k3s)에 Deploy 생성
  • API Gateway: 사용자 요청을 받아 Lambda로 전달
  • DynamoDB (LocalStack): 생성된 Deploy 정보를 로그로 저장
  • IAM: Lambda, S3, DynamoDB 등에 대한 권한 정책 관리
  • SQS: 사용자 API 요청을 큐잉하여 순차 처리, 병렬 처리 시 충돌 방지

6. AWS CLI Container

  • AWS 서비스 프로비저닝 및 관리
  • LocalStack 테스트용 CLI 환경
  • Lambda 패키지 업로드, API Gateway 구성 등 자동화에 활용

7. Veeam Backup (VM Backup)

  • VMware 가상머신의 전체 백업 및 복구 구성
  • 실무형 인프라 운영 환경을 그대로 반영

8. Kubernetes (k3s Cluster)

  • 로컬 서버에서 경량 Kubernetes 클러스터 운영
  • Lambda의 요청을 받아 Deployment 생성
  • Service, ReplicaSet, Pod 관리
  • Dev/Test 환경에서 실무용 컨테이너 오케스트레이션 구성
  • 역할:
  • - 컨테이너 기반 애플리케이션 실행
  • - 멀티 노드(k3s master + worker 구성)
  • - Lambda 기반 자동화와 연동

9. DynamoDB(LocalStack)

  • Lambda에서 생성한 Deploy 로그 저장소
  • 배포 이력 조회 및 상태 모니터링에 사용
  • 완전한 NoSQL 환경을 로컬에서 재현

동작 흐름 (End-to-End)

  1. 사용자가 Cloudflare Access 인증을 통해 웹 페이지 접속
  2. HAProxy 웹서버에서 요청 입력
  3. 요청은 Cloudflare Tunnel → LocalStack API Gateway로 전달
  4. API Gateway는 Lambda 함수 호출
  5. Lambda는 Kubernetes(k3s) API 호출하여 Deployment 생성
  6. 생성된 배포 정보는 DynamoDB(LocalStack)에 저장
  7. 결과 화면이 사용자에게 반환
  8. 모든 정적 파일 및 백업은 MinIO/Veeam을 통해 관리
