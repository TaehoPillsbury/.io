AWS ECS + ALB Path 기반 라우팅 구축

2025 | 개인 학습 프로젝트

프로젝트 개요

Amazon ECS(Elastic Container Service)를 활용하여 여러 컨테이너 애플리케이션을 구축하고, Application Load Balancer(ALB)의 Path 기반 라우팅 기능을 통해 단일 엔드포인트에서 서로 다른 서비스로 트래픽을 분산하는 아키텍처를 구성했습니다.

ECR(Elastic Container Registry)에 저장된 여러 Docker 이미지를 ECS 서비스로 배포하고, /, /mzc/*, /aws/* 경로에 따라 각각 다른 타겟 그룹으로 라우팅하도록 설정했습니다.

주요 구성 요소

네트워크 구성

  • VPC: 프라이빗 서브넷과 퍼블릭 서브넷으로 분리된 네트워크 환경
  • NAT Gateway: 프라이빗 서브넷의 ECS 인스턴스가 외부(ECR 등)와 통신하기 위한 NAT 게이트웨이 구성
  • Internet Gateway: 퍼블릭 서브넷의 ALB가 인터넷과 통신하기 위한 구성
  • 보안 그룹: ALB와 ECS 태스크 간의 트래픽 규칙 설정

컨테이너 오케스트레이션

  • ECS Cluster: EC2 기반 클러스터 구성 (비용 효율성을 위해 EC2 타입 선택)
  • ECS Service: 여러 서비스 정의 및 배포
  • Auto Scaling Group: ECS 컨테이너 인스턴스 자동 확장/축소 설정
  • ECR: Docker 이미지 저장 및 버전 관리

로드 밸런싱

  • Application Load Balancer: Path 기반 라우팅 구성
  • Target Groups: 각 서비스별 타겟 그룹 설정
  • Listener Rules: 경로별 라우팅 규칙 정의

구현 세부사항

Path 기반 라우팅 구성

  • / → 메인 웹 애플리케이션 타겟 그룹
  • /mzc/* → MZC 서비스 타겟 그룹
  • /aws/* → AWS 관련 서비스 타겟 그룹

ECS 서비스 구성

  • 각 서비스는 독립적인 태스크 정의로 구성
  • ECR에서 이미지를 Pull하여 컨테이너 실행
  • 헬스 체크를 통한 자동 복구 기능 활성화

자동 스케일링

  • CPU 및 메모리 사용률 기반 자동 스케일링 정책 설정
  • 최소/최대 인스턴스 수 제한으로 비용 관리
  • ECS 서비스 레벨 스케일링도 별도 구성

학습 목표 및 성과

  • AWS 컨테이너 오케스트레이션 서비스(ECS)의 실제 활용 방법 이해
  • ALB의 고급 라우팅 기능을 통한 마이크로서비스 아키텍처 패턴 학습
  • 프라이빗/퍼블릭 서브넷 분리를 통한 보안 베스트 프랙티스 적용
  • Auto Scaling을 통한 인프라 자동화 및 비용 최적화
  • ECR과 ECS의 통합을 통한 CI/CD 파이프라인 구성 기반 마련

기술 스택
