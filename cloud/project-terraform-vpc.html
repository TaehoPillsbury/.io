Terraform 기반 VPC & 네트워크 모듈

2025 | 개인 인프라 랩

프로젝트 개요

Infrastructure as Code(IaC) 원칙에 따라 Terraform을 활용하여 AWS 네트워크 인프라를 모듈화하고 재사용 가능한 구조로 설계했습니다. 학습 목적이지만 실무에서 사용할 수 있는 수준의 모듈 구조를 목표로 개발했습니다.

VPC, 서브넷, 라우팅 테이블, NAT Gateway, Internet Gateway, 보안 그룹 등을 각각 독립적인 모듈로 분리하여 유지보수성과 재사용성을 높였습니다.

모듈 구조

VPC 모듈

  • CIDR 블록, DNS 설정, 테넌시 등 VPC 기본 설정 관리
  • 가용 영역(AZ) 기반 서브넷 생성 로직 포함

Subnet 모듈

  • 퍼블릭/프라이빗 서브넷 자동 생성
  • CIDR 계산 및 가용 영역 매핑
  • 서브넷 태그 자동 할당

NAT Gateway 모듈

  • NAT Gateway 또는 NAT Instance 선택 가능
  • 비용 최적화를 위한 옵션 제공 (프리티어 환경 고려)
  • Elastic IP 자동 할당

Route Table 모듈

  • 퍼블릭/프라이빗 라우팅 테이블 분리
  • IGW 및 NAT Gateway 라우팅 자동 구성

Security Group 모듈

  • 포트별 인바운드/아웃바운드 규칙 관리
  • 재사용 가능한 보안 그룹 템플릿 제공

Remote State 백엔드

Terraform의 상태 관리를 위해 S3와 DynamoDB를 활용한 Remote State 백엔드를 구성했습니다.

  • S3: Terraform state 파일 저장
  • DynamoDB: State locking을 통한 동시 작업 충돌 방지
  • 버전 관리 및 백업 기능 활용

버전 관리 및 협업

  • GitHub/GitLab을 통한 코드 버전 관리
  • Pull Request 기반 코드 리뷰 프로세스
  • Terraform 코드 스타일 가이드 준수
  • 모듈별 독립적인 테스트 및 배포

학습 목표 및 성과

  • Terraform의 모듈 시스템을 통한 코드 재사용성 향상
  • Remote State를 통한 상태 관리 및 협업 워크플로우 이해
  • AWS 네트워킹 구성 요소들의 관계 및 설정 방법 학습
  • 실무에서 사용 가능한 수준의 인프라 코드 구조 설계 경험
  • Git 기반 코드 리뷰 및 버전 관리 프로세스 경험
