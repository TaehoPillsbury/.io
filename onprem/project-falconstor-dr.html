FalconStor NSS 기반 ESXi DR 복구 프로젝트

온프레미스 환경 | 실무 경험 | 도곡동 본사 ESXi → 삼각지 DR센터 ESXi 복구

프로젝트 개요

고객사 도곡동 본사에 위치한 VMware ESXi 환경에서 장애가 발생하여 VM 데이터 접근이 불가능해진 상황에서, FalconStor NSS(Next-Generation Storage Server)를 활용하여 DR(Disaster Recovery) 센터인 삼각지 ESXi 서버로 볼륨을 복구한 업무 수행 사례입니다.

FalconStor NSS의 Block-level Replication 및 Snapshot 기능을 활용해 빠르게 데이터 접근성을 회복하고 서비스 중단 시간을 최소화한 것이 핵심 목표였습니다.

프로젝트 목표

  • 도곡동 ESXi에서 장애가 발생한 VM 볼륨을 식별
  • FalconStor NSS 백업 스토리지에 저장된 LUN/Volume 스냅샷 확보
  • 삼각지 DR 센터의 ESXi 서버로 해당 볼륨을 재매핑
  • VMware에서 VM 복구 및 정상 부팅 확인
  • 서비스 다운타임 최소화 및 데이터 무결성 검증

시스템 구조

Primary Site (도곡동 본사)

  • VMware ESXi Cluster
  • FalconStor NSS Production Node
  • FC(Fibre Channel) 기반 Storage Volume
  • 운영 중인 주요 VM(업무 서버, DB, AD 포함)

DR Site (삼각지 DR센터)

  • VMware ESXi 별도 DR Cluster
  • FalconStor NSS DR Node
  • Replication Volume 존재
  • DR 네트워크(Tunnel 또는 MPLS)를 통한 동기/비동기 백업

주요 기술 요소

  • FalconStor NSS Volume Snapshot
  • Block-level Replication
  • FC(Fibre Channel) LUN Mapping
  • VMware ESXi Datastore Resignature & VMX 재등록

복구 절차

1. 장애 상황 분석

  • 도곡동 ESXi의 특정 Datastore에서 I/O Hang 현상 발생
  • VM 전원이 꺼지는 문제 및 VMDK 접근 불가
  • 스토리지에서 대상 LUN이 정상적으로 mount되지 않음
  • DR 복구 절차 진행 결정

2. FalconStor NSS 백업 스냅샷 확인

  • FalconStor NSS 관리 콘솔 접속
  • 해당 ESXi datastore가 포함된 Volume 선정
  • 최근 성공적으로 생성된 Snapshot/Replica 확인
  • DR ESXi에 매핑 가능한 상태임을 검증

3. DR(삼각지) 환경으로 Volume 재매핑

수행 내용:

  • DR 사이트에 있는 FalconStor NSS 장비에서 Replica Volume 선택
  • ESXi DR 서버에 FC Target / LUN Mapping 구성
  • ESXi에서 rescan하여 Datastore 인식 여부 확인

결과:

  • DR ESXi에서 Datastore가 정상적으로 인식됨
  • 필요 시 VMware에서 Resignature Datastore 작업 수행 (중복 UUID 방지)

4. VMware VM 복구 작업

  • DR ESXi Datastore에서 기존 VM 폴더 탐색
  • 손상되지 않은 VMX 파일을 기반으로 VM 재등록
  • vNIC, CPU, Memory 설정 복구
  • Snapshot이 있을 경우 Consolidation 수행
  • VM 전원 On → 부팅 테스트

5. 데이터 무결성 검증

  • OS 정상 부팅 여부 확인
  • 주요 서비스(웹, DB, AD 등) 기동 테스트
  • DR 홍보망/업무망에서 통신 검증
  • 사용자 데이터 최신 상태 확인
  • 응용 프로그램 레벨 데이터 검증(로그·트랜잭션 등)

프로젝트 성과

  • ESXi 장애 발생 후 1시간 이내에 DR 운영환경 확보
  • FalconStor NSS 기반 Snapshot/Replica 기술 능숙하게 사용
  • 고객사의 핵심 업무 서비스 중단 시간 최소화
  • VMware + Storage + DR 구성을 end-to-end로 복구 성공
  • DR 매뉴얼 문서화 및 고객사 인수인계 완료

  • VMware ESXi Datastore Resignature 및 VM 복구
  • DR 운영 프로세스 설계 및 트러블슈팅
  • 실제 장애 대응 경험 기반 문제 해결 능력
