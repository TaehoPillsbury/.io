AIX – FalconStor Network Storage Service – HP Storage 기반 고가용성(HA) & DR 구성 작업

온프레미스 환경 | 실무 경험 | Brocade SAN 조닝 → Storage 볼륨 매핑 → Network Storage Service Snapshot 보호 → AIX 미러링 → 장애 복구까지 End-to-End 구축

작업 개요

IBM Power 시스템(AIX OS 기반)의 핵심 업무 스토리지 환경을 고가용성(High Availability) 및 DR(Disaster Recovery)에 대응할 수 있도록 구성한 사례입니다.

HP Storage의 엔터프라이즈급 LUN을 FalconStor Network Storage Service로 중계 및 보호하고, AIX 서버에는 HP Storage 원본 볼륨과 Network Storage Service 보호 볼륨을 미러링(Mirroring)하여 장애 발생 시에도 서비스가 지속될 수 있도록 설계했습니다.

또한 장애 시점의 Network Storage Service Snapshot을 기반으로 장애 시점 이전의 정상 데이터를 AIX에 재할당하여 신속하게 원복할 수 있는 체계를 구축하였습니다.

구성도

작업 목표

  • SAN Storage 기반 엔터프라이즈 HA 구조 설계 및 구축
  • HP Storage → FalconStor Network Storage Service → AIX 간 End-to-End 데이터 흐름 구성
  • Snapshot 기반 데이터 보호 및 복구 체계 구축
  • 미러링 기반 무중단 또는 최소중단 장애 대응 체계 구성
  • 스토리지 장애 시 AIX 운영환경을 신속하게 복구하도록 설계

구성 절차

1. Brocade SAN 스위치 구성 (WWN Zoning)

수행 작업

  • HP Storage, FalconStor Network Storage Service, IBM AIX 서버의 HBA WWN 수집
  • Brocade Fabric OS 접속
  • Single Initiator – Multiple Target 방식으로 Zone 구성
  • Zone → ZoneSet 활성화(Effective)

결과

  • AIX ↔ Network Storage Service ↔ HP Storage 간 SAN 연결이 정상적으로 형성됨
  • WWN 조닝으로 스토리지 보안 및 구조적 안정성 확보

2. HP Storage 볼륨(LUN) 생성 및 Network Storage Service로 매핑

수행 작업

  • HP Storage에서 신규 LUN 생성
  • LUN 사이즈, RAID 레벨, 패리티 구성
  • FalconStor Network Storage Service Host Mapping 수행
  • Network Storage Service에서 LUN 인식 확인

결과

  • HP Storage 원본 LUN이 FalconStor Network Storage Service에서 Block Device 형태로 정상 인식됨

3. FalconStor Network Storage Service 볼륨 보호 정책(Snapshot) 설정

수행 작업

  • Network Storage Service에서 해당 볼륨을 Protection Group에 포함
  • Snapshot 스케줄 정의(예: 1시간마다, 24시간 보관)
  • Block-level Snapshot/Replica 활성화
  • Snapshot 자동 증분 모드(Incremental Forever) 설정

결과

  • HP Storage 원본 볼륨이 Network Storage Service에서 Snapshot 기반으로 보호됨
  • 장애 시 특정 시점으로 복구 가능해짐

4. 보호된 Network Storage Service 볼륨을 AIX 서버로 할당

수행 작업

  • Network Storage Service에서 AIX WWN을 선택하여 볼륨 매핑
  • SAN 스위치 조닝 already 완료 상태
  • AIX에서 cfgmgr로 신규 디스크 인식
  • LSPV를 통해 Disk Identification 수행

주요 AIX 명령어:

  • cfgmgr - 장치 구성 매니저 실행
  • lsdev -Cc disk - 디스크 장치 확인
  • lspv - 물리적 볼륨 목록 확인

결과

  • AIX에서 Network Storage Service 제공 LUN이 hdisk 형태로 정상 인식됨

5. AIX: 기존 HP Storage 볼륨 + Network Storage Service 볼륨 미러링 구성

수행 작업

  • 기존 AIX 볼륨(VG) 구성 확인
  • 새로 매핑된 Network Storage Service 볼륨을 동일 Volume Group에 추가
  • Logical Volume 레벨에서 Mirror 생성

AIX 미러링 명령어 예시:

  • extendvg rootvg hdisk2 - Volume Group에 디스크 추가
  • mirrorvg rootvg hdisk1 hdisk2 - Volume Group 미러링
  • syncvg -v rootvg - 볼륨 그룹 동기화

결과

  • HP Storage 볼륨과 Network Storage Service Snapshot-Protected 볼륨이 실시간 미러링 상태가 됨
  • 원본 장애 시에도 Network Storage Service 볼륨으로 운영 가능

6. 장애 발생 시 복구 절차

예시 시나리오: HP Storage 원본 볼륨 또는 Path Fail/Corruption 발생

수행 단계

  1. 장애 감지
  2. Network Storage Service Snapshot 중 장애 이전 정상 시점의 Snapshot 선택
  3. Snapshot을 새로운 LUN으로 생성("Clone from Snapshot")
  4. SAN Zone 유지 상태에서 해당 Snapshot-LUN을 AIX에 추가 매핑
  5. AIX에서 신규 디스크 검색
  6. 기존 장애 Logical Volume에서 문제 디스크 제거
  7. Snapshot 기반 Disk로 자료 재구성 또는 Logical Volume 재확장
  8. 시스템 재가동 후 서비스 정상화

결과

  • 장애 이전의 정상 시점으로 원복 가능
  • 데이터 손실 최소화 (RPO ≈ Snapshot 스케줄)
  • 서비스 중단 시간 최소화 (RTO 매우 짧음)
