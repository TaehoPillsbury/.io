Veeam Backup & Replication + FalconStor VTL 기반 가상 테이프 백업 시스템 구축

온프레미스 환경 | 실무 경험 | Client 서버 백업 데이터를 FalconStor VTL(Virtual Tape Library)로 이관하는 엔터프라이즈 백업 아키텍처 구축

작업 개요

고객사 운영 서버(Client Server)들의 백업 데이터를 더 안정적이고 장기 보관 가능한 구조로 만들기 위해 Veeam Backup & Replication과 FalconStor VTL(가상 테이프 라이브러리)을 연동한 백업 시스템을 구축한 사례입니다.

SAN 기반 FC(Fibre Channel) 환경에서 VTL 볼륨을 클라이언트 서버에 직접 할당하고, Veeam을 이용해 Disk → Virtual Tape → 장기 보관(Archive) 구조를 구성하여 기존 고객사의 백업 체계를 엔터프라이즈급으로 확장하는 것을 목표로 했습니다.

작업 목표

  • Veeam 백업 솔루션과 FalconStor VTL을 연동하여 중앙집중식 백업 환경 구축
  • Client 서버가 가상 Tape Library를 직접 인식할 수 있도록 SAN Zoning 구성
  • 장기 보관 백업 아키텍처(D2D2T: Disk → Disk → Tape) 구현
  • 백업 데이터 암호화 및 무결성 유지
  • 백업 성능 및 안정성 향상

구성 절차

1. SAN Zoning 구성 (Brocade/Fabric 환경)

수행 작업

  • Client 서버 HBA WWN 확인
  • FalconStor VTL 서버 WWN 확인
  • SAN 스위치에서 아래 구조로 Zone 구성
  • Single Initiator – Single Target 방식 권장
  • Zone 활성화 (ZoneSet Enable)

결과

  • Client Server ↔ FalconStor VTL 간 Fibre Channel 통신 경로 확보
  • OS 수준에서 VTL 장비가 물리 Tape Library처럼 인식될 준비 완료

2. FalconStor VTL 구성

주요 작업

  • VTL Library 생성
  • Virtual Tape Drive 수량 설정
  • Slot 용량(테이프 개수) 구성
  • 자동 Tape Rotation 정책 설정
  • FC Target 포트 활성화
  • Client 서버 WWN을 선택해 VTL Library 매핑

결과

  • Client 서버가 물리 Tape Library처럼 가상 테이프 드라이브와 카트리지를 인식할 수 있는 상태 확보

3. Client 서버에서 VTL 인식 테스트

AIX/Linux/Windows OS 공통 고려사항

  • OS에서 FC 스캔
  • Tape Library 장비 인식(dmesg or device manager 확인)
  • Medium Changer, Tape Drive 정상 등록 확인

결과

  • OS에서 FalconStor VTL이 정상적으로 "테이프 장비"로 인식됨
  • Veeam에서 해당 테이프 장비를 선택해 Job 생성 가능

4. Veeam Backup & Replication 연동

구성 내용

  • Veeam Console에서 Tape Infrastructure 메뉴 진입
  • Add Tape Server → Client 서버 선택
  • Add Tape Library → FalconStor VTL 자동 검색
  • Backup Job 생성 (Disk → Tape Job)
  • GFS Backup 정책 적용 (Weekly / Monthly / Yearly)

결과

  • 운영 서버 백업 → VTL로 자동 이관되는 D2D2T 백업 구조 완성
  • 장기 보관(LT0) 정책 적용 가능

5. 백업 정책 설정

백업 종류 정책
Daily Backup Disk Backup
Weekly Backup Tape Full Backup
Monthly Archive Virtual Tape Export(자동)
Long-term Retention VTL Tape Vaulting 적용

장점

  • 랜섬웨어나 서버 장애 발생 시 테이프 기반 백업으로 빠르게 복구 가능
  • Snapshot + Tape 구조로 복구 시나리오 다중 확보

장애/복구 테스트 수행

  • 백업 데이터 Tape Restore 테스트
  • 특정 시점 파일 복구(File-level Restore)
  • Tape Library Slot 이동(Mount/Unmount) 테스트
  • 실제 백업 볼륨 크기 증가 시 자동 카트리지 할당 테스트

결과: 모든 회차 Job 성공, 실제 운영 환경에서 안정적으로 백업 수행됨
