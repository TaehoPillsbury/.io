▼ 자기 소개

인프라를 직접 구성하고 수정하며 배우는 걸 좋아합니다. 온프레미스 환경에서 엔터프라이즈 백업 솔루션(FalconStor, Veeam)
SAN 스위치 조닝, 서버 하드웨어 유지보수 및 OS 운영 경험을 쌓았으며,
VMware와 LocalStack을 활용하여 로컬 환경에서 3-Tier 아키텍처를 구축하고
AWS 상에서 3-Tier 아키텍처, VPC/NAT/OpenVPN, ECS + ALB + ECR, GitLab & GitHub를 이용한 Terraform 기반 자동 배포 등을
실습하고 있습니다.

기술 스택

  • AWS: VPC, Subnet, IGW, NAT Gateway/Instance, EC2, ALB, ECS(EC2 & Fargate), ECR, S3, CloudFront, DynamoDB
  • Infra as Code: Terraform (VPC, EC2, SG, OpenVPN, ECS, ALB 등 모듈 구성 및 remote state)
  • CI/CD: GitHub Actions, GitLab CI, Docker 기반 GitLab Runner
  • 네트워크 & 보안: VPC Peering, NAT, OpenVPN, 보안 그룹, 라우팅 테이블 트러블슈팅
  • 가상화 & 로컬 개발: VMware 17.5 Pro, LocalStack(AWS 서비스 에뮬레이션), Docker
  • 온프레미스 인프라: FalconStor Backup, Veeam Backup, Brocade SAN Switch 조닝, Dell EMC 서버 유지보수, AIX 운영 및 트러블슈팅

도곡동 본사 ESXi 장애 발생 시 FalconStor NSS의 Snapshot Replica를 활용하여
삼각지 DR센터 ESXi 서버로 신속하게 Volume을 매핑하고 VM 복구를 수행했습니다.

  • DR ESXi 서버로 FC LUN 재매핑 및 Datastore 복구
  • VMware VM 재등록 및 부팅 성공, 데이터 무결성 검증
  • RTO 약 40분 내 복구 달성, 서비스 중단 시간 최소화
FalconStor NSS VMware ESXi Disaster Recovery Fibre Channel Block-level Replication

AIX – FalconStor NSS 스냅샷 볼륨 기반 장애 복구

온프레미스 환경 | 실무 경험

IBM Power 시스템(AIX OS)의 핵심 업무 스토리지를 예상치 못한 장애에 대응할 수 있도록
Brocade SAN 스위치 조닝부터 Storage 볼륨 매핑, NSS Snapshot 보호, AIX 미러링까지 End-to-End로 구축했습니다.

  • Brocade SAN 스위치 WWN 기반 Zone 구성
  • HP Storage → FalconStor NSS → AIX 간 데이터 흐름 구성
  • NSS Snapshot 기반 데이터 보호 및 복구 체계 구축
  • AIX 미러링을 통한 무중단 장애 대응 체계 구현, RTO 20~30분 달성
AIX FalconStor NSS HP Storage Brocade SAN High Availability

Veeam Backup & Replication + FalconStor VTL 기반 가상 테이프 백업 시스템 구축

온프레미스 환경 | 실무 경험

Client 서버 백업 데이터를 FalconStor VTL(Virtual Tape Library)로 이관하는
엔터프라이즈 백업 아키텍처를 구축했습니다. SAN 기반 FC 환경에서 D2D2T(Disk → Disk → Tape) 구조를 구성했습니다.

  • Brocade SAN Zoning을 통한 Client 서버 ↔ FalconStor VTL 연동
  • Veeam Backup & Replication과 VTL 연동, Tape Job 구성
  • D2D2T 백업 아키텍처 구축 및 GFS 백업 정책 적용
  • 장기 보관(LT0) 정책 구현 및 복구 테스트 성공
Veeam Backup FalconStor VTL Brocade SAN D2D2T Tape Backup
